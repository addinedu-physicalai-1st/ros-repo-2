# SC-40: QR 스캔으로 상품 추가

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` 내부 (SM 변경 없음) |
| **트리거** | [📦 담기] → QR 스캔 성공 → 장바구니 추가 |
| **우선순위** | HIGH |
| **연관** | UR-30, SR-60, SR-61, SR-62 |

---

## 사전 조건

- SM = `TRACKING` 또는 `TRACKING_CHECKOUT`
- 테스트용 QR 코드 출력 준비 (예: `{"name": "콜라", "price": 1500}` 인코딩)

## 시험 절차

1. **[앱] [📦 담기] 클릭 → QR 스캔 패널(3-E) 열기**
   - 스마트폰 카메라가 활성화되어 뷰파인더 표시

2. **스마트폰 카메라로 상품 QR 코드 스캔**
   - 웹앱 jsQR 디코딩 → `qr_scan` 소켓 이벤트 → control_service
   - `add_cart_item("콜라", 1500)` → DB CART_ITEM 추가 (`is_paid=0`)
   - `cart` 이벤트 push

3. **[앱] 장바구니 목록 갱신 확인**
   - "콜라 1,500원" 항목 표시 (WebSocket `cart` 이벤트)
   - 타이머 30초 리셋 확인

4. **추가 QR 스캔** (감자칩 등)
   - 스캔마다 목록 갱신

5. **[담기 완료] 클릭 또는 30초 타임아웃 → 패널 닫기**

6. **SM 상태 유지 확인**
   ```bash
   ros2 topic echo /robot_54/status
   ```

## 예상 결과

- SM 변경 없음 (`TRACKING` 유지)
- CART_ITEM DB 추가 확인 (`is_paid=0`)
- [앱] 메인 쇼핑 패널 장바구니에 추가 항목 표시
- 합계 금액 갱신
