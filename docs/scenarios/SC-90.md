# SC-90: 다중 로봇 경로 충돌 → RMF 자동 협상

| 항목 | 내용 |
|---|---|
| **상태 전환** | 두 로봇 동시 `GUIDING` / `RETURNING` — SM 전환 없음 |
| **트리거** | RMF Traffic Schedule이 경로 충돌 감지 → 한 로봇 대기 |
| **우선순위** | MEDIUM |
| **연관** | SR-44 |

---

## 사전 조건

- 로봇 #54, #18 모두 기동 (`TRACKING` 상태)
- `shoppinkki_rmf` Fleet Adapter 기동
- Nav2 기동, 초기 위치 설정 완료 (두 로봇 모두)
- 두 로봇의 GUIDING 목적지가 동일 레인을 공유하는 위치로 설정

## 시험 절차

1. **두 앱에서 동시에 상품 안내 요청**
   - 앱 #54: "베이커리" 검색 → navigate_to (zone_id=7)
   - 앱 #18: "음료" 검색 → navigate_to (zone_id=6)
   - 두 목적지가 동일 중앙 통로(j_mid 경유)를 공유

2. **RMF Traffic Schedule 충돌 감지 확인**
   - 두 TaskRequest가 동시 제출됨
   - RMF가 레인 그래프 상에서 경로 충돌 감지

3. **한 로봇 대기 확인**
   - #54 또는 #18 중 하나가 현재 위치에서 정지
   - SM은 `GUIDING` 유지 (BT4 Nav2 Goal 전송 보류)
   ```bash
   ros2 topic echo /robot_54/status   # mode: GUIDING
   ros2 topic echo /robot_18/status   # mode: GUIDING
   ```
   - 이동 중인 로봇과 대기 중인 로봇 구분 (맵 오버레이 또는 RViz)

4. **선행 로봇 통과 후 후행 로봇 출발 확인**
   - 선행 로봇이 충돌 구간을 통과하면 RMF가 후행 로봇에 navigate 재개 명령
   - 두 로봇 모두 목적지 도착

5. **각 로봇 WAITING 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status   # mode: WAITING
   ros2 topic echo /robot_18/status   # mode: WAITING
   ```

## 예상 결과

- 두 로봇이 순차적으로 목적지에 도착 (동시 충돌 없음)
- 두 앱 모두 "도착" 팝업 표시
- 각 로봇 SM = `WAITING`

## 변형 테스트

- 두 로봇 동시 `RETURNING` (SC-60 상황 × 2): P1/P2 슬롯 각각 배정되어 충돌 없이 주차
