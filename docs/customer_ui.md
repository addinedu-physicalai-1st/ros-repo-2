# Customer UI 명세

> **기술 스택:** Flask + SocketIO (포트 8501). 스마트폰 브라우저용 웹앱.
> **통신:** 채널 A (WebSocket, 브라우저 ↔ customer_web) + 채널 C (TCP, customer_web ↔ control_service)

---

## 구현 기능 목록

| # | 기능 | 관련 시나리오 | 진입 조건 |
|---|---|---|---|
| 1 | 세션 확인 / 로그인 | S01, S12 | 앱 최초 진입 |

## 화면 구성

### 1. `login.html` — 로그인

**역할:** 로봇 QR 스캔 또는 URL 진입 시 표시. 사용자 ID/PW 입력.

**UI 구성:**
```
┌─────────────────────┐
│   🛒 ShopPinkki     │
│                     │
│  아이디 [________]  │
│  비밀번호 [_______] │
│                     │
│    [ 로그인 ]       │
│                     │
│  ⚠ 오류 메시지 영역 │
└─────────────────────┘
```

**UI 요소:**
- 아이디 / 비밀번호 입력 필드
- [로그인] 버튼 — 클릭 시 채널 C `{"cmd": "login", ...}` 전송
- 오류 메시지 영역: "잘못된 아이디 또는 비밀번호", "이미 사용 중인 카트" 등

**플로우:**
```
진입 → session_check (채널 C)
    ├─ 세션 없음 → 로그인 폼 표시
    ├─ 세션 있음 (내 로봇) → main.html 리다이렉트
    └─ 로봇 사용 중 → blocked.html 리다이렉트

로그인 시도 → login (채널 C)
    ├─ 성공 → main.html 리다이렉트
    └─ 실패 → 오류 메시지 표시
```

---

### 2. `blocked.html` — 중복 사용 차단

**역할:** 이미 다른 사용자가 해당 로봇을 사용 중일 때 표시.

**UI 구성:**
```
┌─────────────────────┐
│   🚫 사용 불가      │
│                     │
│  현재 다른 사용자가 │
│  이 카트를 이용 중  │
│  입니다.            │
│                     │
│  [ 새로고침 ]       │
└─────────────────────┘
```

**UI 요소:**
- 안내 메시지
- [새로고침] 버튼 — session_check 재시도

---

### 3. `main.html` — 메인 쇼핑 화면

**역할:** 로그인 후 전체 쇼핑 세션 동안 사용하는 단일 페이지. 로봇 상태(mode)에 따라 패널이 전환된다.

#### 3-A. 공통 상단 바

```
┌──────────────────────────────────────┐
│ 🤖 #54  TRACKING  🔋72%   [🗺️ 맵]  │
└──────────────────────────────────────┘
```
- 로봇 번호, 현재 모드 뱃지 (색상 구분), 배터리 %
- **[🗺️ 맵]** 버튼 — 클릭 시 맵 오버레이(3-I) 표시. 모든 mode에서 항상 활성화
- 1~2Hz WebSocket push로 자동 갱신

#### 3-B. 인형 등록 패널 (mode=REGISTERING)

```
┌─────────────────────────────┐
│  📷 인형을 카메라에          │
│     보여주세요               │
│                             │
│       ⟳ 감지 중...          │
│                             │
│  [ ━━━━━━━━━━━━━ ]  0%     │
└─────────────────────────────┘
```
- 안내 문구 + 스피너 (WebSocket `registering` 수신 시 표시)
- 등록 완료(`registration_done` 수신) 시 → TRACKING 패널로 전환

#### 3-C. 추종/대기 패널 (mode=TRACKING 또는 WAITING)

```
┌─────────────────────────────┐
│  [🔍 상품 찾기]  [📦 담기]  │
├─────────────────────────────┤
│  장바구니                    │
│  • 콜라       1,500원  [✕] │
│  • 감자칩     2,000원  [✕] │
│  ──────────────────────     │
│  합계         3,500원       │
├─────────────────────────────┤
│  [⏸ 잠깐 멈춰]  [🚪 종료]  │
└─────────────────────────────┘
```

- **[상품 찾기]** — 검색 패널 열기 (3-D)
- **[담기]** — QR 스캔 패널 열기 (3-E) → `{"cmd": "mode", "value": "ITEM_ADDING"}`
- **장바구니** — WebSocket `cart` 수신마다 갱신. 각 항목 [✕] 클릭 → `{"cmd": "delete_item", "item_id": N}`
- **[잠깐 멈춰]** — WAITING 전환. `{"cmd": "mode", "value": "WAITING"}` (TRACKING 상태에서만 표시)
- **[따라와]** — TRACKING 복귀. `{"cmd": "mode", "value": "TRACKING"}` (WAITING 상태에서만 표시)
- **[쇼핑 종료]** — `{"cmd": "mode", "value": "RETURNING"}`. 장바구니에 상품 있으면 확인 다이얼로그 먼저

#### 3-D. 상품 검색 패널 (mode=TRACKING, 오버레이)

```
┌─────────────────────────────┐
│  🔍 상품 검색               │
│  [________________] [🎤]   │
│  [ 검색 ]  [ 닫기 ]         │
│                             │
│  → "음료 코너으로 안내합니다" │
└─────────────────────────────┘
```
- 텍스트 입력 또는 STT (마이크 버튼)
- [검색] → `{"cmd": "find_product", "product": "콜라"}` → LLM 경유 zone 반환
- 결과 수신(`find_product_result`) → "X 코너로 안내합니다" 메시지 + Nav2 이동 시작 (검색 패널 닫힘)
- 도착(`arrived`) 수신 → 검색 패널 닫힘, **도착 팝업(3-H)** 표시

#### 3-H. 도착 팝업 (mode=GUIDING → WAITING 전환 시점, 모달)

```
┌─────────────────────────────┐
│                             │
│  📍 도착했습니다!            │
│                             │
│  [상품명] 코너에             │
│  도착했습니다.               │
│                             │
│        [ 확인 ]             │
│                             │
└─────────────────────────────┘
```
- `arrived` WebSocket 이벤트 수신 시 모달 팝업으로 표시 (배경 어둡게)
- 로봇은 WAITING 상태로 전환 (현재 위치에서 정지 대기)
- **[확인]** 클릭 → `{"cmd": "mode", "value": "TRACKING"}` 전송 → TRACKING 복귀
- 팝업이 열려있는 동안 메인 쇼핑 패널은 비활성화

#### 3-E. QR 스캔 패널 (mode=ITEM_ADDING, 오버레이)

```
┌─────────────────────────────┐
│  📷 상품 QR을 스캔하세요     │
│                             │
│  • 콜라  1,500원 ✓          │
│  • 감자칩 2,000원 ✓         │
│                             │
│  (30초 무활동 시 자동 종료)  │
│  [ 담기 완료 ]              │
└─────────────────────────────┘
```
- 스캔 성공마다 목록 갱신 (WebSocket `cart` 수신)
- [담기 완료] 또는 30초 타임아웃 → TRACKING 복귀

#### 3-F. 결제 패널 (mode=CHECK_OUT)

```
┌─────────────────────────────┐
│  💳 결제                    │
│                             │
│  • 콜라       1,500원       │
│  • 감자칩     2,000원       │
│  ──────────────────────     │
│  합계         3,500원       │
│                             │
│  [ 결제하기 ]               │
│  [ 계속 쇼핑하기 ]          │
└─────────────────────────────┘
```
- BoundaryMonitor 결제 구역 진입 시 자동 표시 (WebSocket `status` mode=CHECK_OUT 수신)
- [결제하기] → `{"cmd": "process_payment"}` → 성공 시 RETURNING, 실패 시 ALARM
- [계속 쇼핑하기] → `{"cmd": "mode", "value": "TRACKING"}`

#### 3-G. 알람 패널 (mode=ALARM)

```
┌─────────────────────────────┐
│  🚨 알람 발생               │
│                             │
│  [알람 유형별 메시지]        │
│                             │
│  PIN: [_ _ _ _]             │
│  [ 해제 ]                   │
└─────────────────────────────┘
```

| 알람 유형 | 메시지 | 색상 |
|---|---|---|
| THEFT | 경계 구역을 벗어났습니다. 카트를 마트 내부로 이동해주세요. | 빨강 |
| BATTERY_LOW | 배터리가 부족합니다. 잠시 대기해 주세요. | 노랑 |
| TIMEOUT | 오랫동안 반응이 없어 대기 중입니다. | 주황 |
| PAYMENT_ERROR | 결제에 실패했습니다. 다시 시도해주세요. | 보라 |

- PIN 입력 후 [해제] → `{"cmd": "dismiss_alarm", "pin": "1234"}` (THEFT/BATTERY_LOW/TIMEOUT)
- PAYMENT_ERROR → PIN 없이 [다시 결제] 버튼

#### 3-I. 맵 오버레이 (모든 mode에서 접근 가능, 전체화면 모달)

```
┌─────────────────────────────┐
│  🗺️ 마트 지도        [ ✕ ] │
├─────────────────────────────┤
│                             │
│   ┌──────────────────────┐  │
│   │  [맵 이미지]         │  │
│   │                      │  │
│   │   ● (내 로봇 #54)    │  │
│   │                      │  │
│   │   ● (로봇 #18)       │  │
│   │                      │  │
│   └──────────────────────┘  │
│                             │
│  ● 내 로봇  ● 다른 로봇     │
└─────────────────────────────┘
```

**UI 요소:**
- 상단: 제목 + [✕] 닫기 버튼
- 맵 영역: `shop.pgm` 기반 SVG/Canvas 이미지 위에 로봇 위치 마커 오버레이
  - **내 로봇 (●, 파란색)**: `my_robot.pos_x / pos_y`를 맵 픽셀 좌표로 변환하여 표시
  - **다른 로봇 (●, 회색)**: `other_robots[]`의 각 로봇 위치 마커. 상태는 표시하지 않음
- 하단 범례: 파란색 = 내 로봇, 회색 = 다른 로봇 (색상 설명만)
- 맵 이미지는 핀치 줌 지원 (모바일 스마트폰 대응)

**동작:**
- `status` WebSocket 이벤트 수신마다 마커 위치 및 색상 자동 갱신 (1~2Hz)
- GUIDING 중에도 실시간 갱신 — 로봇이 이동하는 경로를 맵 위에서 확인 가능
- 맵 오버레이를 열어도 백그라운드 동작(GUIDING, WAITING 등) 유지
- [✕] 또는 오버레이 외부 터치 → 맵 닫기

**좌표 변환 (Canvas 픽셀 기준):**
```
px_x = (pos_x - origin_x) / resolution
px_y = img_height_px - (pos_y - origin_y) / resolution   # Y축 반전
```
> `resolution`, `origin`은 `shoppinkki_nav/maps/shop.yaml`에서 읽어 초기화.

## 유저 플로우

```
[스마트폰 QR 스캔 / URL 접속]
        ↓
  session_check
  ├─ 로봇 사용 중 → blocked.html (새로고침 대기)
  └─ 비어있음 → login.html
        ↓
  ID / PW 입력 → 로그인
  ├─ 실패 → 오류 메시지 재입력
  └─ 성공 → main.html
        ↓
  ┌─ [인형 등록 패널] ─────────────────────────────┐
  │  인형을 카메라에 보여줌 → YOLO 감지 → 등록 완료 │
  └─────────────────────────────────────────────── ┘
        ↓
  ┌─ [메인 쇼핑 패널: TRACKING] ───────────────────────────────────────────┐
  │                                                                       │
  │  [🗺️ 맵] (상단 바, 항상 접근 가능)                                   │
  │      → 맵 오버레이 표시 (내 로봇 + 다른 로봇 위치, 실시간 갱신)       │
  │      └─ [✕] → 맵 닫기 (현재 mode 유지, 백그라운드 동작 계속)         │
  │                                                                       │
  │  [담기] → QR 스캔 모드 → 상품 스캔 → [담기 완료] → TRACKING 복귀     │
  │                                                                       │
  │  [찾기] → 텍스트/STT 검색 → 안내 시작 (GUIDING)                      │
  │      │     맵 오버레이로 실시간 이동 확인 가능                         │
  │      └─ 도착 팝업(WAITING) → [확인] → TRACKING 복귀                  │
  │                                                                       │
  │  [잠깐 멈춰] → WAITING 패널                                           │
  │      └─ [따라와] → TRACKING 복귀                                      │
  │                                                                       │
  │  결제 구역 진입 (자동) → [결제 패널]                                  │
  │      ├─ [결제하기] → 성공 → 복귀 대기열                              │
  │      ├─ [계속 쇼핑] → TRACKING 복귀                                  │
  │      └─ 결제 실패 → [알람 패널] → PIN 해제 → WAITING                 │
  │                                                                       │
  │  알람 발생 (자동) → [알람 패널] → PIN 해제 → 이전 상태 복귀           │
  │                                                                       │
  │  [쇼핑 종료] → RETURNING → 대기열 배정 → 로봇 대기 위치 이동          │
  └───────────────────────────────────────────────────────────────────── ┘
        ↓
  로봇 STANDBY → 사용자 카트 수령 → 세션 종료 → login.html
```

## WebSocket 이벤트 처리 요약

| 수신 이벤트 | UI 동작 |
|---|---|
| `status` (mode 변경) | 모드 뱃지 갱신. mode에 따라 패널 전환. 맵 오버레이가 열려있으면 `my_robot` / `other_robots` 마커 위치·색상 갱신 |
| `cart` | 장바구니 목록 갱신 |
| `registering` | 등록 패널 스피너 표시 |
| `registration_done` | 메인 쇼핑 패널로 전환 |
| `find_product_result` | "X 코너로 안내합니다" 표시 |
| `arrived` | 도착 팝업(3-H) 표시. 로봇 WAITING 전환. [확인] 클릭 → TRACKING 복귀 |
| `nav_failed` | "안내에 실패했습니다" 토스트 |
| `alarm` | 알람 패널 전환 (event_type별 메시지) |
| `payment_done` | 결제 완료 메시지 → RETURNING 패널 |
